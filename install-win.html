    Установка под Windows

    TeX

    • Установите MikTeX
    • Установите шрифты (скачайте, распакуйте, выберите в Проводнике все файлы и нажмите правой кнопкой Установить):
      • Computer Modern Unicode (cm-unicode). Выберите все TTF файлы.
      • Latin Modern Math (lm-math) Нужный вам файл в папке OTF.
    • Установите Python 2, если у вас его нет. Третий не засчитывается! Пропишите его в PATH: Windows 7/8: Win+Break -> Дополнительные параметры системы -> Дополнитлеьно -> Переменные среды. Выберите PATH, допишите в него точку с запятой и путь до папки установки Python 2 (обычно C:\Python27), а также ещё раз точку с запятой и подпапку Scripts. В итоге, например, нужно дописать 
      ;C:\Python27;C:\Python27\Scripts
    • Выполните из консоли, находясь в папке C:\Python27\Scripts 
      easy_install pygmets

    build.cmd
    Если у вас есть bash, можете вызывать build.sh, под Windows он тоже корректно отработает. Её можно поставить для сборки из Far по нажатию Enter: F9 -> Commands -> File Assotiations -> F4; маска *.tex, комманда – скрипт. Аналогично, можно настроить эту команду в TeXworks в меню Правка -> Настройка -> Вёрстка.

    Git

    Если вы предпочитаете работу с визуальным приложением, которое, к сожалению, не даст хорошей настраиваемости, но с которым легко работать, ставьте GitHub for Windows. В ином случае, ставьте консольный Git.

    Где работать.

    Собственно, набирать TeX рекомендуется в TeXworks или Far (тут уже с настройками). Для просмотра PDF во время работы рекомендую Preview от TeXworks, поскольку эта программа не будет мешать менять PDF файл при вёрстке и git pull. Для работы с консольным Git рекомендуется Far или аналогичный файловый менеджер с доступом в консоль (Total Commander,...), поскольку консоль Windows сама по себе не так удобна.
